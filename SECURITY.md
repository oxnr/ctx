# Security Policy

This repository is primarily a static site, curriculum, and repo-local tooling project.

Please do not post active secrets, exploit payloads, or credential material in public issues or pull requests.

## Reporting

If you find a security issue:

1. Use GitHub private vulnerability reporting if it is enabled for the repository.
2. If private reporting is unavailable, open a minimal public issue without exploit details and request a private follow-up channel.
3. If credentials or tokens were exposed, revoke and rotate them before reporting.

## Good Reports

Include:

- affected file, page, script, or workflow
- impact and realistic abuse path
- reproduction steps without destructive payloads
- suggested remediation if you have one

## Out Of Scope

- theoretical issues with no practical impact
- generic dependency CVEs when the dependency is not used in production here
- spam, credential stuffing, or social engineering reports without repo-specific evidence
